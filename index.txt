We have covered relational databases using SQL & SQLite3

We dealt with CRUD operation using SQL language model (DDL, DML, DQL)

We have an abastraction layer: SQLAlchemy, which is an ORM
    => SQLAlchemy is your contract with the database 
    — it defines what data looks like, 
    how tables relate, 
    and what rules the database enforces 

Extensions: Flask-SQLAlchemy (Object Relational Mapper)

    => Define databases table as Python Classes
    => Interact with your database using Python methods and objects instead of writing SQL statements manually.

The session — your atomic workspace

The session is a staging area. 
    Think of it as a draft payment. You fill in all the details — 
    add objects, change values, stage deletions — 
    but nothing reaches the database until you commit. 
    If anything goes wrong before commit, rollback cancels everything as if nothing happened.

This is ACID's A — Atomicity. Every staged change commits together as a unit, or none of them do.
Method	                    Parameters	What it does

db.session.add(obj)	        obj:        model instance	Stage one object for INSERT or UPDATE
db.session.add_all(list)	list:       list of model instances	Stage multiple objects at once
db.session.commit()	        none	    Write all staged changes to the database
db.session.rollback()	    none	    Cancel all staged changes since last commit
db.session.delete(obj)	    obj:        model instance	Stage one object for DELETE
db.session.get(Model, id)	Model:      class, id: pk value	Fetch one row by primary key — None if missing
db.session.flush()	        none	    Send changes to DB without committing — gets assigned IDs
db.session.refresh(obj)	    obj:        model instance	Re-read object from database — discards stale in-memory state
db.session.expire(obj)	    obj:        model instance	Mark object as stale — next access re-fetches from DB


Query: Every query method with parameters
Method	            Returns	          Use when

.all()	            list	          You want every matching row
.first()	        object or None	  You want the first match — safe, never raises
.one()	            object	Exactly   one result expected — raises if 0 or 2+
.one_or_none()	    object or None	  Zero or one expected — raises only if 2+
.count()	        int	              You just need the number — no objects loaded
.filter_by(**kwargs)Query	          Exact equality filters — simple and readable
.filter(*criteria)	Query	          Comparisons, LIKE, IN, OR, complex logic
.order_by(*cols)	Query	          Sort results — chain desc() for descending
.limit(n)	        Query	          Cap result count — always combine with order_by
.offset(n)	        Query	          Skip n rows — combine with limit for pagination
.get(id)	        object or None	  Fetch by primary key — checks identity map first

Universal field parameters — apply to every field type:
Parameter	    Type	        What it does

required	    bool	            True = error if field missing from load() input
dump_only	    bool	            True = appears in responses, silently ignored on input. Use for id, timestamps.
load_only	    bool	            True = accepted in requests, never in responses. Use for passwords.
allow_none	    bool	            True = null/None is valid. Default False.
validate	    callable or list	Validator function(s) to run after type check
load_default	any	Value           used when field is missing from load() input
dump_default	any	Value           used when attribute is None during dump()
data_key	    str	JSON            key name when it differs from Python attribute name
metadata	    dict	            Extra info — used by OpenAPI/Swagger documentation generators
